# v3.0.1 — v3 device fixes and auth polish

**Status:** Complete.

## Scope

### Player-facing

- Standalone Android APK (`npm run build:apk`) launches without crashing on open
- Auth error messages are plain language (e.g. invalid email → “Enter a valid email address.”)
- Sign-up with email confirmation shows **Check your email** (not “Sign up failed”)
- Feedback form validates optional contact email before submit

### Engineering

- Align `expo-secure-store`, `expo-auth-session`, and `expo-web-browser` to Expo SDK 54 (was SDK 57)
- Add `expo-secure-store` and `expo-web-browser` config plugins in `app.json`
- App entry `index.js` imports `react-native-gesture-handler` before Expo Router
- Break `syncService` ↔ store circular import via `pushIfSignedIn.ts`
- `authErrors.ts`, `authValidation.ts`, and `presentAuthMessage` for consistent auth UX
- Defensive try/catch around auth session init
- Unit tests for auth error mapping and validation (`authErrors.test.ts`, `authValidation.test.ts`); 69 tests total

## Acceptance criteria

### APK / release build

- [x] Preview APK installs and opens to the welcome screen
- [x] Expo Go dev workflow unchanged (`npm run start:tunnel`, `npm run start:test:tunnel`)

### Auth UX

- [x] Invalid or malformed email shows “Enter a valid email address.” (not raw Supabase text)
- [x] Wrong password shows “Incorrect email or password.”
- [x] Duplicate sign-up shows “An account with this email already exists. Try signing in.”
- [x] Email confirmation required after sign-up shows info alert and routes to sign-in
- [x] Google, email sign-in, and sign-up use the same friendly error presentation

### Feedback

- [x] Invalid optional contact email shows a friendly validation message
- [x] Submit failures do not expose raw database errors

## Out of scope

- Sign in with Apple, leaderboards, remote push (see [backlog.md](./backlog.md))
