# v3.0.3 — Google OAuth PKCE and themed messages

**Status:** Complete.

## Scope

### Player-facing

- Google sign-in on standalone Android APK completes with a local session (not just a Supabase user record)
- Auth errors, sign-up validation, and feedback use themed in-app modals instead of native Android alerts

### Engineering

- Supabase client uses `flowType: 'pkce'` so OAuth returns `?code=` query params (Android deep links do not preserve URL hash fragments)
- `parseAuthRedirectUrl` reads both query and hash redirect params; session exchange retries when a session already exists
- OAuth callback waits for in-flight Google sign-in and skips duplicate code exchange
- `AppMessageHost` + `presentAppMessage` replace native `Alert` for auth, sign-up validation, and feedback messages (styled like `GameModal`)
