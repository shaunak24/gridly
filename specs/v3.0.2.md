# v3.0.2 — Google OAuth deep link fix

**Status:** In progress.

## Scope

### Player-facing

- Google sign-in on standalone Android APK completes without an “unmatched route” error
- After Google OAuth, the app returns to the home screen with the user signed in

### Engineering

- `app/auth/callback.tsx` handles `gridly://auth/callback` deep links from Supabase OAuth
- `handleAuthCallback` skips code exchange when a session already exists (avoids double-processing on Android)
- `signInWithGoogle` checks for an existing session when the browser session dismisses but the deep link succeeded

## Acceptance criteria

### Google OAuth (APK)

- [ ] Google sign-in from welcome screen completes and lands on home (no “page could not be found”)
- [ ] Google sign-in from sign-in screen completes and lands on home
- [ ] Google sign-in from guest profile menu completes and lands on home
- [ ] User appears signed in (profile menu shows account, not guest)
- [ ] Email sign-in and sign-up still work unchanged
