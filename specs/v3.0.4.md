# v3.0.4 — Sign-out themed confirmation modal

**Status:** Complete.

## Scope

### Player-facing

- Sign-out confirmation in the profile menu uses a themed in-app modal with **Sign out** and **Cancel** buttons (not a native Android alert)

### Engineering

- `GameModal` and `AppMessageHost` support optional primary/secondary actions for confirm dialogs
- `ProfileMenu` sign-out uses `presentAppMessage` instead of `Alert.alert`

## Acceptance criteria

- [x] Tapping **Sign out** in the signed-in profile menu shows a themed modal
- [x] Modal offers **Sign out** and **Cancel**; cancel dismisses without signing out
- [x] Confirming sign-out returns the user to the welcome screen

## Out of scope

- Migrating remaining game/settings `Alert.alert` calls (see [backlog.md](./backlog.md) if promoted later)
