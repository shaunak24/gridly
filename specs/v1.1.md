# v1.1

## Goal

Extend Gridly beyond MVP with daily play, persistence, sharing, hard mode, theming, settings, an interactive tutorial, and a daily puzzle reminder.

## Status

**Complete** — ready for device validation via Expo Go (SDK 54). Local notifications require a development build.

## Features

### Daily puzzle and practice

- [x] Home offers **Play daily** / **Continue daily** and **Practice** / **Continue practice**.
- [x] **Daily** uses one shared word per calendar day (device local timezone, midnight reset).
- [x] Completing the daily puzzle locks it until the next local day.
- [x] **Practice** picks a random word with no daily lock.
- [x] In-progress daily and practice games **resume** after leaving the game screen or restarting the app.

### Stats

- [x] Games played, win rate, current streak, max streak, guess distribution (1–6 and losses).
- [x] Persisted locally via AsyncStorage.
- [x] **Stats** screen linked from home.

### Share grid

- [x] After a win, user can copy an emoji grid to the **clipboard**.
- [x] Share text includes puzzle number and score without revealing the answer word.
- [ ] Native share sheet (WhatsApp, Messages, etc.) — planned v1.2 ([changelog.md](./changelog.md)).

### Hard mode

- [x] Toggle on **Settings** screen.
- [x] When enabled, guesses must honor all revealed hints.

### Theme

- [x] Theme options: **Dark**, **Light**, **System** (Settings).
- [x] Light-bulb icon on home toggles dark ↔ light quickly.
- [x] Persisted locally; applies across all screens.

### Settings

- [x] Gear icon on home opens dedicated **Settings** screen.
- [x] Hard mode, daily reminder toggle, reminder time picker, theme cycle.

### Daily reminder

- [x] Optional local notification at a user-selected time (default **8:00 AM** device local time).
- [x] Requests OS permission when enabled.
- [x] **Expo Go:** scheduling may fail; user sees an alert. Works in a **development build**.

### How to play

- [x] Interactive tutorial with speech-balloon steps.
- [x] Guided example **REACT → BRAVE → CRANE**; explains teal, amber, and slate.
- [x] Compact board layout while keyboard is visible.

### Tile animations and post-game UX

- [x] Two-sided sequential flip animation on submitted guesses.
- [x] Keyboard hides after win or loss; result and actions use extra space.
- [x] Win/loss modal delayed ~2 seconds after the final guess.
- [x] Dismissible modal without redundant helper text.

## Out of scope (v1.1)

- Native share to other apps (v1.2)
- Haptics (v1.2)
- User accounts and cloud sync (v2)
- Leaderboards (v2)
- Remote push notifications (v2)
