# v3.1 — Clickable Word Hunt share invites

**Status:** Complete.

## Scope

### Player-facing

- **Create puzzle** — share and copy HTTPS invite links that are clickable in WhatsApp and other chat apps
- **Open invite** — tapping a shared link opens a landing page, then the custom puzzle in Gridly
- **Legacy links** — old `gridly://…?code=g1:…` custom puzzle links still work

### Engineering

- `game_invites` Supabase table with public read for active invites
- `create-invite` and `resolve-invite` Edge Functions
- `src/platform/invites/` — generic invite types, URL builder, service, share message
- `EXPO_PUBLIC_INVITE_LINK_BASE` optional env (defaults from `EXPO_PUBLIC_SUPABASE_URL`)

## Acceptance criteria

### Create and share

- [x] User enters a valid 5-letter word and taps **Share puzzle** — system share sheet opens with HTTPS link
- [x] **Copy link** copies the HTTPS invite URL to clipboard with confirmation modal
- [x] Shared HTTPS link is clickable in WhatsApp (manual APK test)
- [x] Guest (not signed in) can create and share an invite
- [x] Signed-in user can create an invite (`created_by` populated server-side)

### Open invite

- [x] Tapping HTTPS link on device with Gridly opens custom puzzle with correct word
- [x] Landing page shows when link opened in browser; redirects to app via `gridly://`
- [x] Invalid or expired invite shows friendly error and returns to Word Hunt hub
- [x] Legacy `gridly://games/word-hunt/play?mode=custom&code=g1:…` still opens puzzle
