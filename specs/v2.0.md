# v2.0

## Goal

Transform Gridly from a single word puzzle into a **multi-game platform** with a game-card hub, while preserving Word Hunt behavior and introducing Grid Snap as a second grid-based game.

## Status

**In progress**

## Features

### Platform home

- [x] Gridly branding with tagline “Grid-based games.”
- [x] Game cards for **Word Hunt** and **Grid Snap**
- [x] Theme quick-toggle and **app settings** (theme only) on platform home

### Word Hunt (migrated)

- [x] Game hub at `/games/word-hunt` with daily, practice, create puzzle, stats, how to play
- [x] Play screen at `/games/word-hunt/play` — behavior identical to v1.2
- [x] Word Hunt settings (hard mode, daily reminder) on game settings screen
- [x] In-game Home → Word Hunt hub; hub back → platform home
- [x] Legacy `gridly://game?...` links redirect to Word Hunt play

### Grid Snap (new)

- [x] Game hub at `/games/grid-snap` with daily, practice, stats, how to play, settings (mirrors Word Hunt structure)
- [x] Random image puzzles via Picsum (optional Pexels when `EXPO_PUBLIC_PEXELS_API_KEY` is set)
- [x] Difficulty: Easy 4×4, Medium 6×6, Hard 8×8
- [x] Drag pieces; matching neighbors snap together into groups (no tile animation in v2.0; polish deferred)
- [x] Grouped pieces move together after snap
- [x] Win when all pieces are in solved positions
- [x] Grid Snap settings: default difficulty for practice

### Engineering

- [x] `src/platform/gameRegistry.ts` — extension point for new games
- [x] Per-game modules under `src/games/<id>/`
- [x] Shared theme and app settings under `src/shared/`
- [x] Namespaced storage keys with v1.x migration
- [x] `puzzleEngine` unit tests for Grid Snap

## Out of scope (v2.0)

- User accounts and cloud sync
- Haptics
- Grid Snap share, move timer, image caching polish
- See [v2.1.md](./v2.1.md) for full backlog
