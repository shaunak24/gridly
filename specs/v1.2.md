# v1.2

## Goal

Polish post-game UX, improve share content, add a daily countdown, and let users create and share custom word puzzles via deep links.

## Status

**Complete**

## Features

### Post-game modal and end bar

- [x] Win modal shows a celebration emoji; loss modal shows a sympathetic emoji.
- [x] Daily win/loss modals are **dismiss-only** — no Practice or Play again button in the modal.
- [x] After dismissing the modal, the end bar below the board offers **Practice** for daily games.
- [x] Practice games show **Play again** in the end bar and modal.
- [x] Completed daily puzzles cannot be restarted from the game screen (guard redirects to home).

### Share content

- [x] Share text includes the puzzle date, attempt count (e.g. `3/6`), and the answer word.
- [x] Emoji grid rows are unchanged.

### Daily countdown

- [x] When today's daily is complete, the home **Play daily** button shows a live countdown on the right (`MM:SS` or `H:MM:SS`).
- [x] The Play daily button remains disabled while the countdown is visible.

### Custom puzzles

- [x] Home offers **Create puzzle** — user enters a valid 5-letter word.
- [x] App generates a `gridly://` share link encoding the word.
- [x] **Share puzzle** opens the system share sheet with message and deep link.
- [x] Opening the link starts a custom game with the encoded word.
- [x] Custom games do not affect daily completion or stats.

## Out of scope (v1.2)

- Native share sheet
- Haptics
- User accounts and cloud sync
- In-app feedback / bug report (backlog — revisit after cloud storage)
