# v3.0 — Cloud accounts, sync, and feedback

**Status:** Complete — shipped; device fixes and auth polish in [v3.0.1.md](./v3.0.1.md).

## Scope

### Player-facing

- Optional user accounts (email/password, Google)
- Welcome screen on first launch; guest play via **Continue as guest**
- Profile menu on platform home (guest or signed-in; modal with account actions)
- Cross-device sync of stats and settings when signed in
- App settings: theme, send feedback
- Per-game daily reminders for Word Hunt and Grid Snap (each with its own toggle and time; default on at 8:00 AM)
- In-app feedback and bug report form

### Engineering

- Supabase Auth + Postgres with row-level security
- Local-first stores with cloud push on change and merge on sign-in
- Supabase Edge Function for feedback email notifications (Resend)
- SQL migration in `supabase/migrations/001_v3_initial.sql`

## Acceptance criteria

### Accounts

- [x] Guest can play all games without signing in
- [x] Welcome screen offers sign in, Google, create account, and continue as guest
- [x] Profile menu shows guest state or signed-in email; sign out returns to welcome
- [x] Email/password sign-up and sign-in work with Supabase configured
- [x] Google OAuth works in development build (`npm run build:apk`)
- [x] Sign out restores local-only data from device storage

### Sync

- [x] Word Hunt stats, Grid Snap stats, game settings, and app theme sync to cloud when signed in
- [x] Sign-in merges local and cloud data (stats summed, max streaks, latest settings by `updated_at`)
- [x] In-progress games stay local only
- [x] Stats screens show a non-blocking “Sign in to sync” hint when guest

### Reminders

- [x] Word Hunt settings: daily reminder toggle and time picker
- [x] Grid Snap settings: daily reminder toggle and time picker
- [x] Each game schedules its own local notification
- [x] Default: both reminders on at 8:00 AM for new installs
- [x] Existing users who disabled Word Hunt reminders keep that preference

### Feedback

- [x] App settings links to feedback screen
- [x] User can submit feedback or bug report
- [x] Submission stored in Supabase `feedback` table
- [x] Edge Function sends email notification when configured

## Out of scope

- Leaderboards, remote push, in-progress game sync across devices, Sign in with Apple (see [backlog.md](./backlog.md))

## Setup (developer)

1. Create a Supabase project
2. Run `supabase/migrations/001_v3_initial.sql` in the SQL editor
3. Copy `.env.example` to `.env` and set `EXPO_PUBLIC_SUPABASE_URL` and `EXPO_PUBLIC_SUPABASE_ANON_KEY`
4. Set the same `EXPO_PUBLIC_*` vars in EAS for APK builds
5. Enable Email auth with **Confirm email OFF** for development (see [supabase/README.md](../supabase/README.md))
6. Run `migrations/002_user_profile_trigger.sql` after the initial migration
7. Configure Google OAuth redirect: `gridly://auth/callback`
8. Deploy `supabase/functions/send-feedback-notification` and set secrets: `RESEND_API_KEY`, `FEEDBACK_TO_EMAIL`
9. Add a database webhook on `feedback` insert → Edge Function

See [supabase/README.md](../supabase/README.md).
